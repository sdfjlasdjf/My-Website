Projects

1. Rental System (Apartments, Roommates, and Furniture)

I designed a rental system that allows users to post and find apartments, roommates, or secondhand furniture, and also chat directly with the owner. For this project, I used Spring Boot as the main backend framework and Vue 3 for the frontend. Redis was utilized for user authorization and caching, while MySQL served as the primary database. The system follows RESTful conventions for basic CRUD operations.

Users can either log in or register on the login page; upon registration, a token is generated, stored in ThreadLocal and Redis, and then sent back to the frontend.

After logging in, a user selects a category (Finding a Place, Finding Roommates, or Secondhand Furniture), enters a location, and sees all relevant posts. Clicking on a post opens its detail page, where the user can “like” it or start a direct chat with the owner. Users can also create new posts by clicking on a floating button in the bottom-left corner.

In the profile section, users can edit their personal information or view all posts they have created or liked across the three categories.

2. Mini Spring Writing

I am designing and implementing a lightweight Spring-like framework from scratch, replicating key features such as **bean creation, initialization, and dependency injection**. This project involves developing custom annotations like @ComponentScan, @Component, @Scope, and @Autowired, enabling component discovery and injection. Additionally, I implemented a **BeanPostProcessor** to introduce custom logic before and after bean initialization. The framework also integrates **Aspect-Oriented Programming (AOP)** to support **method interception and cross-cutting concerns**, allowing for modularized and maintainable code. Currently, I am working on enhancing Spring MVC by simplifying annotations for specific use cases, making development more streamlined.

2. Chat System with Rich Features

I am building a chat system that allows users to send emojis, voice messages, and even transfer money. There is also a camera feature to send pictures directly. This project is still under active development, and I'm continually adding new features.

Technologies used:

  • Go as the main programming language
  • Gin Framework for building RESTful APIs quickly
  • WebSocket for real-time communication
  • GORM for ORM and database interaction
  • Goroutines and Channels to handle concurrent tasks efficiently

One of the core advantages of Go is its lightweight concurrency model using goroutines and channels. This makes it easy to handle many simultaneous chat connections, ensuring responsive real-time messaging even under heavy load.

3. Restaurant Suggestion System

This was my first major project and served as my senior design project at NYU (completed as a group). The system recommends restaurants based on the user’s location and dish-style preferences, which are identified through a simple chatbot. This chatbot is not powered by advanced AI; rather, it parses user sentences and extracts key terms to find relevant dining options.

We also included a feature to send calendar notifications and email reminders about a selected restaurant. This functionality was implemented using APIs for email sending and schedule management.

Overall, it was a challenging yet enlightening project that introduced me to user interaction design, geolocation services, integrating external APIs, and implementing basic CRUD operations in a team setting.
